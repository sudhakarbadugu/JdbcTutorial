PreparedStatement:

		Statement		- Parent interface
			|			- extends
	PreparedStatement	- child interface

- It is an interface. It is child of Statement.
- It improves the performance. 
- PreparedStatement uses the positional parameters. 
- When ever we using the positional paramters, database will identify the query from query cache and returns the results fast.
- positional parameters index starts from 1.

Important methods:
-----------------
	1. executeQuery()
	2. executeUpdate()
	3. seXxx(int indexPosition, value)


What will happen without positional parameters?
- Query will send to database server. data base server is a program. it will save all tables data into a file called .dbf.
- It will validate the query. If query is good then it runs the algorithm and finds the table information. This informaion will be saved in a query cache. If same query runs again, db server will skip this step.

- If query is wrong, it will return error.
- If query changes each time, then it runs above st
ep again.


What is positional parameter?
	- Changing the real value to ? called position parameter.
	Eg:	Original query
		select * from stock_user where username ='murali';
	to (positional parameter query)
		select * from stock_user where username = ?;


When we should use PreparedStatement?
- When query is repeadly changing the parameters, then we should use the PreparedStatement. Here parameters are getting changed so these type queries we should use PreparedStatement.
	Eg: select * from stock_user where username ='murali';
	Eg: select * from stock_user where username ='prasad';



- If we use the positional parameter and if we are not setting positional parameter value then below exception will occur
java.sql.SQLException: Missing IN or OUT parameter at index:: 1

Transaction management:
Saving one or more tables into database in a signle transaction called transaction management.
Eg:	If an user order a laptop and payment is failed, then we should mark order as failed and should add records to both tables.

- If order is success then we should save the records to both tables.


How will you achive transaction management in jdbc?
1. We need to disable the autocommit as shown below. By default in jdbc autoCommit is true.
	connection.setAutoCommit(false);
	- If autoCommit is true, immediately indivisual queries will saved.

			ResultSet rs = stmt.executeQuery("select username, id from stock_user");
			ResultSetMetaData metaData = rs.getMetaData();

			System.out.println(dbMetaData.getDriverName());
			System.out.println(dbMetaData.getJDBCMajorVersion());
			System.out.println(dbMetaData.getSQLKeywords());

		}
	}

			Class.forName("oracle.jdbc.driver.OracleDriver");
